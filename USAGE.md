# 鍵管理 Bot 使い方ガイド

部室の鍵の貸し借りを Discord で記録するための Bot です。
**ボタンをポチッと押すだけ**で「誰が・いつ・何をしたか」が記録されます。

---

## 基本の流れ

操作はぜんぶ Discord のチャンネル内の **ボタン** で行います。
コマンドは特殊な機能のときだけ使います。

```
[借りる] → [開ける] → ……作業中…… → [閉める] → [返す]
```

### ボタン一覧

| ボタン | 意味 | 押すタイミング |
|-------|------|--------------|
| 🟢 **借りる** | 部室の鍵を借りた | 学生課などで鍵を受け取ったとき |
| 🟢 **開ける** | 部屋を開けた | 部室の鍵を開けたとき |
| 🟢 **閉める** | 部屋を閉めた | 部室を出て鍵を閉めたとき |
| 🔴 **返す** | 鍵を返却した | 学生課などに鍵を返したとき |
| 🔵 **受け取る** | 別の人から鍵を受け取った | 部員間で鍵を引き継いだとき |
| ⚪ **持ち出す** | 部室を閉めて鍵を別の場所に持って行く | 別の場所で作業するときなど |
| ⚪ **取り消す** | 直前の操作を取り消す | 押し間違えたとき（1 分以内） |

> ボタンは **常に最新のメッセージにだけ** 表示されます。
> 古いメッセージはログとして残りますが、ボタンは消えています。

---

## 操作シナリオ

### 普通に部室を使う場合

1. 学生課で鍵を借りる → **借りる** を押す
2. 部室の鍵を開ける → **開ける** を押す
3. 作業する
4. 部室を出て鍵を閉める → **閉める** を押す
5. 学生課に鍵を返す → **返す** を押す

### 押し間違えたとき

操作した直後 **1 分以内** に **取り消す** ボタンが表示されます。
押すと直前の操作がなかったことになり、1 つ前のボタンが復活します。

- ⚠️ **本人の操作しか取り消せません**（他人の操作は取り消せない）
- ⚠️ **1 分過ぎるとボタンが消えます**

### 鍵を別の人に渡したとき

- 渡された側が **受け取る** を押してください
- これで持ち主が新しい人に切り替わります（リマインドの送信先も変わります）

### 部室を閉めて、鍵を別の場所に持っていきたいとき

例: 部室を一旦閉めて学生課の隣のラウンジで作業したい場合

1. **持ち出す** ボタンを押す
2. 場所を聞かれるので入力（例: `4 号館 302`）
3. 以降の操作メッセージに「鍵の現在地: 4 号館 302」が表示されます
4. 戻ってきたら **開ける** か、誰かに渡すなら **受け取る**、終わるなら **返す**

「**持ち出す**」ボタンは以下のときに出ます:
- 借りた直後
- 閉めた直後
- 受け取った直後（鍵を引き継いだとき）

部屋が開いている間（**開ける** 押下後）は出ません（鍵は部屋の中にあるはずなので）。

### 金曜日に土日も借りたいとき

平日は学生課が閉まるので返却できないことがあります。
**金曜日だけ**、通常の「借りる」と並んで以下の 2 つが追加表示されます:

- 🟢 **土曜まで借りる**
- 🟢 **日曜まで借りる**

これを押すと、その日まで自動でリマインドが控えめになります（詳細は下の「リマインド」を参照）。

## リマインド機能

鍵を持っている人が長時間返却しないと、Bot が自動で通知してくれます。

### 2 種類のリマインド

| リマインド | デフォルト | 内容 |
|-----------|-----------|------|
| **毎日リマインド** | 20:00 | 「鍵を返し忘れていませんか？」 |
| **無変化リマインド** | 2 時間 | 「○時間 状態に変化がありません」 |

通知は **チャンネルに送信されて、現在の持ち主にメンション**されます。

### 設定変更コマンド

**現在の鍵の持ち主だけ** が変更できます。変更内容はチャンネルに公開通知されます。

| コマンド | 意味 |
|---------|------|
| `/reminder_daily hour:20` | 毎日リマインドの時刻を 20 時に設定 |
| `/reminder_daily hour:0` | 毎日リマインドを停止 |
| `/reminder_idle hours:3` | 無変化リマインドを 3 時間ごとに設定 |
| `/reminder_idle hours:0` | 無変化リマインドを停止 |

> ⚠️ **鍵の持ち主が変わると、設定はデフォルト（20 時 / 2 時間）に自動で戻ります。**
> 例: A さんが「毎日リマインドを 22 時」に変えていても、B さんが受け取ると 20 時に戻ります。

### 状況の確認

`/reminder_status` を叩くと **自分にだけ見える形** で現在の状況が表示されます:

- 現在の鍵の状態（閉まっている / 開いている / 持ち出し中）
- 持ち主
- 鍵の現在地（持ち出し中なら）
- リマインド設定
- 次回送信予定
- 長期貸出（土日まで借りる）の状態

### 長期貸出中のリマインド挙動

「土曜まで借りる」「日曜まで借りる」を押した場合:

| 日 | 毎日リマインド | 無変化リマインド |
|----|--------------|----------------|
| 押した日 | 止まる | 止まる |
| 最終日（土または日） | **20 時に出る** | 止まる |
| 翌日以降 | 通常通り出る | 通常通り出る |

毎日リマインドだけ最終日に通知し、忘れずに返却できるようにしています。

## スラッシュコマンド一覧

### 全員が使えるコマンド

| コマンド | 説明 |
|---------|------|
| `/reminder_status` | リマインド設定と現在の状況を表示（自分にだけ見える） |
| `/nfc_register` | NFC タグ用のトークン発行（再実行で上書き） |

### 持ち主だけが使えるコマンド

| コマンド | 説明 |
|---------|------|
| `/reminder_daily hour:<0-23>` | 毎日リマインドの時刻を変更（0 で停止） |
| `/reminder_idle hours:<0-168>` | 無変化リマインドの時間を変更（0 で停止） |

### 管理者だけが使えるコマンド（デバッグ用）

| コマンド | 説明 |
|---------|------|
| `/debug_friday on:<true/false>` | 金曜モードの ON/OFF を切り替え（土日貸出ボタンの表示テスト） |
| `/debug_reminder type:<daily/idle>` | リマインド通知を即時送信（実際の設定は変わらない） |

## NFC で操作する場合（おまけ機能）

iPhone の Shortcuts と NFC タグを使うと、**スマホをタグにタッチするだけで鍵の開閉を記録**できます。

1. Discord で `/nfc_register` を実行
2. 表示されたトークンをコピー
3. Shortcuts で「URL の内容を取得」を作成
   - URL: `http://<サーバーのIP>:8080/nfc`
   - 方法: POST
   - リクエストボディ (JSON): `{ "token": "貼り付けたトークン" }`
4. Shortcuts を NFC タグに割り当て

タグにタッチするごとに、部室の鍵が「開いた → 閉まった → 開いた → …」とトグルされ、Discord のチャンネルに通知されます。

## よくある質問

### Q. 古いメッセージのボタンが押せない！
A. 仕様です。ボタンは **最新のメッセージにだけ** あります。
   一番下までスクロールするか、最新の操作メッセージを探してください。

### Q. 取り消しボタンが消えた
A. 操作から **1 分過ぎると自動で消えます**。それ以降は取り消せません。
   間違いに気づいたら早めに押してください。

### Q. 「取り消せません」と言われた
A. 取り消せるのは **本人の操作だけ**です。他の人の操作は取り消せません。

### Q. リマインドを止めたい
A. 自分が持ち主のときに `/reminder_daily hour:0` と `/reminder_idle hours:0` の両方を実行してください。

### Q. 鍵を返したのにリマインドが来る
A. 「返す」ボタンを押し忘れていないか確認してください。
   `/reminder_status` で「持ち主: なし（返却済み）」になっていれば OK。

### Q. 自分は持ち主のはずなのに `/reminder_daily` で「持ち主のみ」と言われる
A. 別の人が「受け取る」を押している可能性があります。
   `/reminder_status` で現在の持ち主を確認してください。

### Q. Bot が落ちた / 動かない
A. 管理者に連絡してください。再起動が必要かもしれません。
   状態は JSON ファイルに保存されているので、再起動しても情報は消えません。
